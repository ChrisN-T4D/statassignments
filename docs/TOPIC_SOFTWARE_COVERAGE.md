# Topic content by software

## Layout

- Metadata: [src/data/topicsMeta.js](src/data/topicsMeta.js) (re-exported as `topics` from [src/data/topics.js](src/data/topics.js)).
- Chapter HTML: `src/content/topics/<topicId>__<moduleId>/<software>.html` where `software` is one of `jamovi`, `spss`, `r`, `excel`, `stata`.
- The slug uses `moduleId` so duplicate topic ids (e.g. two `scatterplots` topics) stay unique.
- Runtime resolution: [src/content/topics/resolveTopicHtml.js](src/content/topics/resolveTopicHtml.js) → [src/views/TopicView.vue](src/views/TopicView.vue) with the learner’s preferred software from [src/composables/usePreferredSoftware.js](src/composables/usePreferredSoftware.js).

## How variants are produced

| Track | Source |
|--------|--------|
| jamovi | Original LSJ-derived HTML (extracted from the former monolithic `topics.js`). |
| spss, r, excel, stata (most topics) | Generated by [scripts/adapt-topic-variants.mjs](scripts/adapt-topic-variants.mjs) using [scripts/lib/adaptJamoviHtml.mjs](scripts/lib/adaptJamoviHtml.mjs) (phrase replacements + banner note). |
| spss, r, excel, stata (Module 3: software-interface, data-entry, variable-types) | Built from Vue templates in [src/content/software/module-3/](src/content/software/module-3/) via [scripts/embed-module3-vue-html.mjs](scripts/embed-module3-vue-html.mjs). jamovi for those topics remains the full LSJ chapter. |

## Regenerating HTML

```bash
npm run build:topics
```

This runs extract (no-op if `topics` no longer carry `contentHtml`), re-runs adaptation for all non–Module-3 trio folders, and re-embeds Module 3 from Vue.

After editing a `jamovi.html` file, run `npm run build:topics` so derived tracks stay in sync (except edit Module 3 non-jamovi via the Vue files, then run the same command).

## QA: stray “jamovi” on non-jamovi tracks

Some mentions are intentional (attribution to LSJ, footnotes). To list files with remaining matches:

```bash
rg -l -i jamovi src/content/topics --glob '*__*/spss.html'
```

Repeat for `r.html`, `excel.html`, `stata.html`. Triage any body text that should name the learner’s package instead.

## Regenerating `topicsMeta.js` only

If topic metadata changes in `topics.js` and you temporarily add fields there:

```bash
npm run generate:topics-meta
```

(Normally you edit `topicsMeta.js` or run this after restoring a full `topics` export that includes every field you need to serialize.)
